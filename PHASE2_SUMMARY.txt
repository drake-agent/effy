═══════════════════════════════════════════════════════════════════════════════
  모드별 특성
═══════════════════════════════════════════════════════════════════════════════

MODE: LOCAL (기본값)
├─ 활성화: distributed.enabled: false
├─ 레이턴시: 0-1ms (프로세스 메모리)
├─ 처리량: 매우 높음
├─ 확장성: 단일 프로세스
├─ 사용 사례: 테스트, 개발, 소규모 배포
└─ 특징: Phase 1과 100% 호환

MODE: REDIS
├─ 활성화: distributed.enabled: true, mode: redis
├─ 레이턴시: 5-10ms (네트워크)
├─ 처리량: 높음
├─ 확장성: 수평 확장 (여러 호스트)
├─ 사용 사례: 중규모 배포, 공유 세션
└─ 특징: Redis Pub/Sub + TTL 기반 스토리지

MODE: KUBERNETES
├─ 활성화: distributed.enabled: true, mode: kubernetes
├─ 레이턴시: 10-20ms (K8s DNS)
├─ 처리량: 중간-높음
├─ 확장성: 자동 스케일링 (HPA)
├─ 사용 사례: 프로덕션 규모, 클라우드
└─ 특징: K8s DNS + Circuit Breaker + 자동 발견

═══════════════════════════════════════════════════════════════════════════════
  마이그레이션 경로 (Phase 1 → Phase 2)
═══════════════════════════════════════════════════════════════════════════════

STEP 1: 기본값 유지
└─ distributed.enabled: false
   → 기존 코드 변경 없음, 동작 100% 동일

STEP 2: Local 모드 테스트
└─ distributed.enabled: true
   mode: local
   → 분산 추상화 활용, 프로세스 내 실행

STEP 3: Redis 배포
├─ Redis 서버 시작: docker run -p 6379:6379 redis:latest
├─ 설정: mode: redis, redis.url: redis://localhost:6379
└─ 에이전트 공유 세션 활용

STEP 4: 에이전트 마이크로서비스 분리
├─ 각 에이전트 독립 프로세스: node agent-bootstrap.js --agent=general
├─ Redis로 상태 공유
└─ 독립적 스케일링

STEP 5: Kubernetes 배포
├─ 에이전트 Deployment (replicas: N)
├─ Horizontal Pod Autoscaler (HPA)
└─ 자동 스케일링 + 자가 치유

═══════════════════════════════════════════════════════════════════════════════
  사용 예시
═══════════════════════════════════════════════════════════════════════════════

1. 기본값 (변경 없음)
   → distributed.enabled: false (이미 기본값)

2. Local 테스트
   const { getDistributedArchitecture } = require('./distributed');
   const dist = getDistributedArchitecture(config.distributed);
   await dist.init({});

3. Redis 배포
   const redis = require('redis');
   const client = redis.createClient({ url: 'redis://localhost:6379' });
   const dist = getDistributedArchitecture(config.distributed);
   await dist.init({ redis: client });

4. 마이크로서비스
   const service = await dist.createAgentService(agentInstance, 3101);
   → GET http://localhost:3101/health

5. 메시지 전송
   await dist.messageBus.request('code', 'general', 'analyze', {});

  mode: redis

distributed:
  enabled: true
  mode: kubernetes
  kubernetes:
    namespace: effy
    domain: svc.cluster.local
  agents:
    general: { port: 3101, replicas: 3 }

Effy Phase 2는 단일 프로세스 게이트웨이(Phase 1)를 기반으로 확장 가능한
분산 아키텍처를 제공합니다.

▪ Phase 1과 100% 호환 (기본값)
▪ 3가지 모드 지원 (Local, Redis, Kubernetes)
▪ 프로덕션 품질 (Circuit Breaker, Health Checks)
▪ 점진적 마이그레이션 가능
▪ 자동 스케일링 지원 (K8s HPA)

모든 모듈은 프로덕션 준비 완료 상태입니다.
