# Effy 세션 기억 구조

> Effy가 사용자와의 대화를 어디까지 기억하고, 어떻게 활용하는지 정리한 문서

---

## 한눈에 보기

| 구분 | 저장 위치 | 얼마나 기억? | 언제 사라져? |
|------|----------|-------------|------------|
| **단기 기억** | 서버 메모리 (RAM) | 최대 25번 주고받기 | 30분 동안 대화 없으면 삭제 |
| **대화 기록** | DB (PostgreSQL) | 모든 대화 전부 | 안 사라짐 (영구 저장) |
| **지식 기억** | DB (PostgreSQL) | 대화에서 뽑아낸 사실/결정 | 안 사라짐 (영구 저장) |

---

## 1. 단기 기억 (Working Memory)

**지금 나누고 있는 대화**를 서버 메모리에 들고 있는 것.

### 기본 동작

- 사용자가 말하면 1개, Effy가 답하면 1개 → **1번 주고받기 = 메시지 2개**
- 최대 **50개 메시지 (= 25번 주고받기)** 까지 보관
- **30분 동안 아무 말이 없으면** 세션이 날아감 (다시 말 걸면 새 세션 시작)
- 대화할 때마다 30분 타이머가 리셋됨

### 대화가 길어지면? (자동 요약)

15번 이상 주고받으면(메시지 30개 초과) **자동 요약**이 작동:

```
[1~10번째 주고받기] → LLM이 핵심만 요약 (bullet points로 압축)
[11~15번째 주고받기] → 원문 그대로 유지 (최근 5문답)
```

이렇게 하면 대화가 아무리 길어져도 "이전 맥락 요약 + 최근 대화 원문"으로 기억을 유지할 수 있음.

---

## 2. 대화 기록 (Episodic Memory)

모든 질문-답변이 **DB에 영구 저장**됨. 단기 기억이 사라져도 DB에는 남아있음.

### 어디에 쓰이나?

- **다음 세션에서 과거 대화 참조**: 30분 지나서 단기 기억이 사라져도, DB에서 이전 대화를 검색해서 "아, 이 사람이 전에 이런 얘기 했었지" 하고 참조
- **대시보드 "대화 내역" 탭**: 어드민이 누가 뭘 물어봤는지 확인 가능
- **전문가 찾기**: 누가 어떤 주제에 대해 많이 대화했는지 분석

---

## 3. 지식 기억 (Semantic Memory)

대화 중에 나온 **중요한 사실, 결정, 인사이트**를 따로 뽑아서 저장.

### 예시

- "우리 팀 스프린트는 2주 단위" → 사실(Fact)로 저장
- "React 대신 Vue 사용하기로 결정" → 결정(Decision)으로 저장
- "김대리가 프론트엔드 담당" → 관계(Entity)로 저장

### 어디에 쓰이나?

- 새로운 질문이 오면 관련 지식을 검색해서 답변에 활용
- 예: "프론트엔드 담당 누구야?" → 저장된 지식에서 바로 답변

---

## 4. LLM에 실제로 전달되는 양 (Context Window)

질문의 복잡도에 따라 Effy가 참고하는 정보의 양이 달라짐:

| 복잡도 | 현재 대화 | 과거 대화 | 관련 지식 | 총량 | 예시 |
|--------|----------|----------|----------|------|------|
| **LIGHT** | 5,000 토큰 | 없음 | 없음 | 8K | "안녕", "고마워" |
| **STANDARD** | 10,000 토큰 | 5,000 토큰 | 4,000 토큰 | 35K | "이 기능 어떻게 써?" |
| **DEEP** | 20,000 토큰 | 15,000 토큰 | 8,000 토큰 | 70K | "아키텍처 분석해줘" |

- **현재 대화** (current_thread): 지금 나누고 있는 대화 내용
- **과거 대화** (recent_history): 이전 세션에서 나눈 대화 (DB에서 가져옴)
- **관련 지식** (semantic): 저장된 사실/결정 중 질문과 관련된 것

간단한 인사에는 가볍게, 복잡한 질문에는 더 많은 맥락을 참고해서 답변.

---

## 전체 흐름 예시

```
사용자: "프론트엔드 배포 어떻게 해?"

Effy가 참고하는 것들:
  ├─ 단기 기억: 방금 나눈 대화 5문답 원문
  ├─ 대화 기록 (DB): 지난주에 "배포" 관련해서 나눈 대화 검색
  ├─ 지식 기억 (DB): "배포는 GitHub Actions + ECS" 저장된 사실
  ├─ 사용자 프로필: 이 사람은 AX팀 개발자
  │
  └─ 전부 합쳐서 LLM에 전달 → 맥락 있는 답변 생성
```

---

## 요약

- **단기**: 현재 대화에서 최근 5문답 원문 + 그 이전은 요약문
- **중기**: 30분 이내 대화는 서버 메모리에 보관
- **장기**: 모든 대화가 DB에 영구 저장, 관련 질문 시 검색해서 참조

## 설정 위치 (개발자용)

| 설정 | 파일 | 기본값 |
|------|------|--------|
| 세션 유효시간 | `src/memory/manager.js` → WorkingMemory | 30분 |
| 최대 메시지 수 | `src/memory/manager.js` → maxEntries | 50개 (25문답) |
| 요약 트리거 | `src/memory/manager.js` → summarizeThreshold | 30개 (15문답) |
| 압축 후 유지 | `src/memory/compaction.js` → keepRecentTurns | 10개 (5문답) |
| Context 프로필 | `effy.config.yaml` → budgetProfiles | LIGHT / STANDARD / DEEP |
