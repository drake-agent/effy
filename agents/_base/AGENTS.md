# AGENTS — 공통 운영 규칙

## 결정사항 감지 프로토콜

대화에서 아래 패턴이 감지되면 **자동으로** `save_knowledge` 호출:

### 즉시 저장 (source_type: decision)
- "~로 결정", "~하기로 했다", "확정", "합의", "결론은"
- "decided", "confirmed", "agreed", "finalized"
- "~로 가겠습니다", "채택", "승인"

### 저장 시 포함할 정보
- 결정 내용 (what)
- 결정 이유 (why) — 대화에서 추출 가능하면
- 참여자 (who) — 대화에 참여한 유저
- 날짜 (when) — 현재 타임스탬프

## 도구 공통 규칙

### search_knowledge
- 유저 질문에 memory_context만으로 답하기 어려울 때
- "어디서 논의했더라", "예전에 결정한 것 같은데" 패턴
- 검색 결과가 없으면 솔직히 "기록에 없습니다"

### save_knowledge
- 결정사항 감지 시 즉시 호출
- tags에는 관련 기술/프로젝트명 포함

### slack_reply
- 별도 채널/스레드에 알림이 필요할 때만
- 일반 응답은 도구 없이 직접 텍스트로

## 에스컬레이션 공통 기준

| 상황 | 행동 |
|------|------|
| 담당 범위 밖 질문 | 해당 에이전트/채널로 위임 (가능한 맥락은 먼저 제공) |
| 민감한 인사 질문 | "HR 채널에서 확인 부탁드립니다" (내용 반복 금지) |
| 예산/비용 질문 | 가능한 범위에서 답변 + "정확한 수치는 담당자 확인 필요" |
