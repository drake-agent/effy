================================================================================
                           DESIGN PATTERNS USED
================================================================================

1. Pipeline Pattern (AgentScope-inspired)
   - Composable, reusable steps
   - Multiple execution strategies
   - Clean separation of concerns

2. Builder Pattern (Fluent API)
   - Readable configuration syntax
   - Chainable method calls
   - Type-safe structure building

3. Factory Pattern
   - Pipeline.sequential(), .fanout(), etc.
   - Step registration and lookup
   - Dynamic instantiation

4. Dependency Injection
   - External instances injected to steps
   - Loose coupling with core system
   - Testability

5. Observer Pattern
   - History tracking
   - Event logging
   - Metrics collection

6. Strategy Pattern
   - Different pipeline execution strategies
   - Pluggable processing logic
   - Context-driven routing

================================================================================
                           USAGE EXAMPLES (SUMMARY)
================================================================================

1. Basic Sequential
   Pipeline.sequential()
     .addStep(authStep)
     .addStep(routeStep)

2. Fluent Builder
   PipelineBuilder.create('name')
     .sequential()
       .step(authStep)
     .end()
     .build()

3. Conditional Routing
   Pipeline.conditional()
     .setCondition((ctx) => ctx.isUrgent)
     .whenTrue(urgentHandler)
     .whenFalse(normalHandler)

4. Parallel Execution
   Pipeline.fanout()
     .addStep(notifySlack)
     .addStep(notifyEmail)

5. Retry Logic
   Pipeline.iterative()
     .setStep(fetchData)
     .setCondition((ctx) => ctx.success)
     .setMaxIterations(3)

6. Agent Chain
   Pipeline.agent()
     .addAgent('writer', codeWriterAgent)
     .addAgent('reviewer', codeReviewerAgent)
     .addAgent('documenter', docsAgent)

7. Complex Nested
   PipelineBuilder.create('workflow')
     .sequential('init')
       .step(authStep)
     .end()
     .conditional((ctx) => ctx.severity === 'critical')
       .whenTrue(urgentPipeline)
       .whenFalse(normalPipeline)
     .end()
     .build()

8. YAML Configuration
   pipelines:
     default:
       steps: [auth, route, runtime, save, log]
     urgent:
       steps: [auth, route]
       then:
         conditional:
           field: severity
           critical: [notifyOps, escalate]

================================================================================
                         PERFORMANCE NOTES
================================================================================

Time Complexity:
  - Sequential: O(N) - linear with step count
  - Fanout: O(1) - constant (parallel)
  - Conditional: O(N) - linear (condition + 1 branch)
  - Iterative: O(N×M) - linear × iterations
  - Agent: O(N) - linear with agent count

Space Complexity:
  - Sequential: O(1) - minimal state
  - Fanout: O(N) - stores all results
  - Others: O(1) or O(M) - reasonable bounds

Optimization Tips:
  - Use Fanout for I/O-bound parallel tasks
  - Use Conditional for early exit patterns
  - Use Iterative for retry/polling with limits
  - Nest pipelines for complex workflows
