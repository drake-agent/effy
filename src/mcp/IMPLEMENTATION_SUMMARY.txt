ARCHITECTURE HIGHLIGHTS
─────────────────────────────────────────────────────────────────────────────

1. TRANSPORT LAYER (transport.js)
   - Two implementations: HTTP+SSE and StdIO
   - Common interface: connect(), send(), close()
   - Automatic message ID tracking for request-response
   - Built-in timeout management
   - Event-driven for streaming responses

   HTTP+SSE:
   - Remote servers, firewall-friendly
   - POST for requests, GET/EventStream for responses
   - Suitable for web deployments

   StdIO:
   - Local binaries, low latency
   - Child process stdin/stdout
   - JSONL format (one JSON object per line)

2. REGISTRY LAYER (registry.js)
   - Converts MCP tool definitions to Effy format
   - Maintains server → tools mapping
   - Routes tool calls to correct server
   - Handles tool discovery

   Conversion Features:
   - MCP inputSchema → Effy input_schema
   - Adds category (mcp-<type>)
   - Marks with _mcpServer for internal routing
   - Ensures compatibility with TOOL_DEFINITIONS format

3. CLIENT LAYER (client.js)
   - Orchestrates all MCP servers
   - Parallel initialization (with fallback)
   - Singleton pattern (getMCPClient)
   - Graceful degradation on errors
   - Config-driven from effy.config.yaml

   Lifecycle:
   1. initialize() → Load config
   2. Create MCPServerConnection per server
   3. Each connection: transport.connect() → tools/list
   4. Register tools in MCPToolRegistry
   5. Export via getMCPToolDefinitions()
   6. shutdown() on process exit

INTEGRATION POINTS
─────────────────────────────────────────────────────────────────────────────

1. Configuration (effy.config.yaml)
   ```yaml
   mcp:
     timeout: 30000
     servers:
       - id: github
         transport: sse
         url: http://localhost:3001/sse
         enabled: true
       - id: filesystem
         transport: stdio
         command: npx
         args: ['-y', '@anthropic-ai/mcp-server-filesystem', '/workspace']
         enabled: true
   ```

2. Startup (app.js)
   ```javascript
   const { getMCPClient } = require('./mcp/client');
   const mcpClient = getMCPClient();
   await mcpClient.initialize();
   ```

3. Tool Registry Integration (tool-registry.js)
   ```javascript
   const { getMCPToolDefinitions } = require('../mcp/client');
   const TOOL_DEFINITIONS = {
     // ... existing Effy tools ...
     ...getMCPToolDefinitions(),
   };
   ```

4. Usage in Agents
   ```javascript
   // Direct tool call
   await dispatchTool('github_create_pull_request', {...});
   
   // Or via mcp_call (dynamic)
   await dispatchTool('mcp_call', {
     tool_name: 'github_create_pull_request',
     input: {...}
   });
   ```
